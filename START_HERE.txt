HybridRAG3 -- Start Here
========================

Welcome! This is a document search and question-answering system that
runs on your own computer. You give it a folder of documents and it
lets you ask questions about them in plain English.

You do NOT need to know Python or PowerShell to use this.
Your documents are never modified, moved, or sent anywhere.


STEP 1: Understand what this does (10 minutes)
-----------------------------------------------
Open docs/THEORY_OF_OPERATION_RevA.md in your browser or text editor.
This explains the whole system in plain language with no jargon.

If you see a term you do not understand, check docs/GLOSSARY.md.


STEP 2: Install (30-60 minutes, one-time)
------------------------------------------
Follow docs/INSTALL_AND_SETUP.md from Part 1 through Part 6.

The short version:
  1. Install Python 3.11 from python.org (check "Add to PATH")
  2. Open PowerShell (press Win+X, click Terminal)
  3. Type:  cd "D:\HybridRAG3"
  4. Type:  py -3.11 -m venv .venv
  5. Type:  Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
  6. Type:  .\.venv\Scripts\Activate.ps1
  7. Type:  pip install -r requirements.txt
  8. Edit start_hybridrag.ps1 to set your folder paths
  9. Type:  . .\start_hybridrag.ps1
  10. Type: rag-diag

If step 9 looks odd -- the dot-space-dot at the start is not a typo.
It tells PowerShell to load the commands into your current window.
Without the leading dot, the commands vanish after the script runs.


STEP 3: Put documents in your source folder
--------------------------------------------
Copy PDFs, Word docs, spreadsheets, emails, or any supported files
into the source folder you configured in step 8 above (default:
D:\RAG Source Data). HybridRAG reads these but never changes them.


STEP 4: Index your documents (run once, then only when docs change)
--------------------------------------------------------------------
In the same PowerShell window, type:

  rag-index

First run downloads a small AI model (~87 MB) automatically, then
processes every document. This takes a few hours for a large folder.
After that, re-running rag-index only processes new or changed files.


STEP 5: Ask questions
---------------------
  rag-query "What is the operating frequency?"

Or launch the graphical interface:

  python src/gui/launch_gui.py

The GUI has a dark theme, a query box, an index panel, and a status
bar. See docs/GUI_GUIDE.md for the full walkthrough.


STEP 6 (optional): Set up Ollama or a cloud API for answers
-------------------------------------------------------------
By default, queries use a local AI model via Ollama. If Ollama is not
installed, install it from https://ollama.com, then in a SEPARATE
PowerShell window run:

  ollama serve

Leave that window open. In your HybridRAG window, pull a model:

  ollama pull phi4-mini

For cloud-based answers (faster, requires an API key):

  rag-store-key
  rag-store-endpoint
  rag-mode-online


DAILY USE (after setup is done)
-------------------------------
  1. Open PowerShell (Win+X, Terminal)
  2. Type:  cd "D:\HybridRAG3"
  3. Type:  . .\start_hybridrag.ps1
  4. Type:  rag-query "your question here"

Or double-click start_rag.bat (CLI) or start_gui.bat (GUI) from
File Explorer. See docs/SHORTCUT_SHEET.md, section 2.

COMMON FIRST-TIME PROBLEMS
---------------------------
  "py -3.11 not found"
    Install Python 3.11 from python.org. Check "Add to PATH" during
    the installer. Verify with: py --list

  "Running scripts is disabled"
    Type: Set-ExecutionPolicy -Scope CurrentUser RemoteSigned

  "rag-diag not found"
    You ran start_hybridrag.ps1 without the leading dot.
    Type: . .\start_hybridrag.ps1  (dot space dot backslash)

  "Cannot connect to localhost:11434"
    Ollama is not running. Open a second PowerShell window and type:
    ollama serve

  Queries return "no results"
    You need to index documents first. Run: rag-index
